# /tdd — 测试驱动开发

严格按照 RED → GREEN → REFACTOR 三阶段实施。

## 用法

```
/tdd <功能描述>
```

## 三阶段

### RED — 写失败测试
1. 定义接口契约，编写测试用例（不写实现）
2. 运行测试，确认**预期失败**（原因必须是"未实现"）

### GREEN — 最小实现
1. 写最小代码让测试通过，不做优化
2. 确认全部测试通过

### REFACTOR — 质量提升
1. 消除重复、提升可读性，测试仍通过
2. 函数 < 50 行，文件 < 800 行

## 覆盖率要求

| 代码类型 | 最低覆盖率 |
|----------|-----------|
| 普通业务逻辑 | 80% |
| 认证 / 安全 / 支付 | 100% |
| 题库核心计算 | 100% |
| API 端点 | 90% |

## 测试工具

- **Next.js**：`cd Web && npm test`
- **FastAPI**：`cd Web/backend && poetry run pytest -v`
- 选择器优先 `data-testid`，禁止 CSS 类选择器
